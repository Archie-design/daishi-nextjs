## 優先實作：定時評論監控 + 低分示警

**價值**：第一時間掌握負面評論，快速處理避免影響口碑。

### 技術方案
- 使用 **Google Business Profile API**（商家所有者可存取全部評論）
- GAS 時間觸發器：每 2 小時自動執行一次
- 比對上次掃描時間，找出新評論
- 評分 ≤ 4 星時觸發通知（Email 或 LINE Notify）

### 實作步驟
1. 在 Google Cloud Console 啟用 My Business Business Information API
2. 設定 OAuth 2.0 憑證，取得商家帳號授權
3. 在 GAS 新增 `checkNewReviews()` 函式，呼叫：
   `GET https://mybusiness.googleapis.com/v4/accounts/{accountId}/locations/{locationId}/reviews`
4. 將上次掃描時間存入 ScriptProperties
5. 設定 GAS 時間觸發器定時執行

### 通知內容格式
```
【大師修桃園店】收到新評論
評分：⭐⭐⭐（3 星）
評論者：王小明
內容：「...」
時間：2026-04-06 14:30
→ 前往回覆：https://maps.app.goo.gl/bNskpCdwU1cK88N48
```

## 功能 2：評論歷史記錄 Dashboard（Google Sheets）

**價值**：視覺化口碑趨勢，追蹤長期評分變化。

### 技術方案
- 每次監控掃描時，將評論資料寫入 Google Sheets
- Sheets 自動產生圖表：每週平均評分折線圖、評論數長條圖

### Sheets 欄位設計
| 日期 | 評論者 | 評分 | 評論內容 | 是否已回覆 | 備註 |
|------|--------|------|----------|------------|------|

## 功能 3：負評 AI 回覆草稿（Gemini）

**價值**：節省思考時間，提供有溫度的官方回覆範本。

### 技術方案
- 監控發現 ≤ 4 星評論時，同步呼叫 Gemini API
- 根據評論內容產生 2 則回覆草稿（正式版 / 親切版）
- 草稿附在通知 Email 中，審閱後手動貼到 Google Maps

### Prompt 設計重點
- 代入商家名稱「大師修桃園店」
- 態度誠懇、不辯解、展現解決意願
- 字數控制在 80～150 字

---

## 功能 4：每週評論摘要報告

**價值**：不用主動查，每週自動掌握口碑狀況。

### 報告內容
- 本週新評論數 vs 上週
- 本週平均評分 vs 本月均值
- 最高分評論（可截圖用於行銷）
- 最低分評論（需優先處理）
- Gemini 產生的本週評論一句話摘要

### 觸發時間
每週一早上 9:00 寄送（GAS 時間觸發器）

## 功能 5：店內 QR Code 產生器（後台）

**價值**：方便印製實體引導材料，提升評論收集率。

### 技術方案
- 後台管理頁面新增「產生 QR Code」按鈕
- 使用 Google Charts API 產生 QR Code 圖片（免費，無需額外 API Key）：
  `https://chart.googleapis.com/chart?chs=300x300&cht=qr&chl={編碼後的評論URL}`
- 支援下載為 PNG

## 功能 6：評論完成後引導彈窗（前台）

**價值**：提升顧客實際發布評論的轉換率。

### 互動流程
1. 顧客點「📋 複製」按鈕
2. 彈出全螢幕遮罩，顯示：
   - 「評論已複製！」
   - 「現在前往 Google Maps 貼上，只需 10 秒」
   - 大按鈕：「⭐ 立即前往貼上」→ 開啟 Google Maps 評論頁
   - 小連結：「稍後再去」

## API 限制與注意事項

| 資料來源 | 可取得評論數 | 需求 | 備註 |
|---------|------------|------|------|
| Google Business Profile API | 全部評論 | 商家帳號 OAuth 授權 | **推薦方案** |
| Places API | 最新 5 則 | Google Maps API Key | 有每月免費額度限制 |
| 爬蟲 | 理論上全部 | — | 違反 Google 服務條款，不建議 |

### Google Business Profile API 申請流程
1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 啟用「My Business Business Information API」
3. 建立 OAuth 2.0 用戶端憑證
4. 在 GAS 中使用 `ScriptApp.getOAuthToken()` 或手動 OAuth 流程取得 token
5. 使用 `UrlFetchApp` 呼叫 API
